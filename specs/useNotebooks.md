## Hook: useNotebooks

### 1. Responsibility and Purpose
`useNotebooks` encapsulates all CRUD operations for the `notebooks` collection in IndexedDB. Notebooks act as organisational containers for notes. The hook exposes a reactive list of notebooks and four async mutators. It is planned for a future UI iteration ‚Äî no component currently consumes it.

### 2. Return Value
| Field | Type | Description |
|---|---|---|
| `notebooks` | `Array<Object>\|undefined` | Reactive list sorted alphabetically by `title`. `undefined` while the initial query is in-flight. |
| `createNotebook` | `Function` | Persists a new notebook and returns its UUID. |
| `updateNotebook` | `Function` | Updates mutable fields; never overwrites `id` or `createdAt`. |
| `deleteNotebook` | `Function` | Permanently removes a notebook. Does NOT cascade-delete associated notes. |
| `getNotebookById` | `Function` | Non-reactive point read by UUID. |
| `error` | `Error\|null` | Last caught error; `null` initially. |

### 3. Data Model Invariants
- `id` ‚Äî UUID generated by `crypto.randomUUID()` at creation time; immutable.
- `createdAt` ‚Äî set once at creation; never overwritten by `updateNotebook`.
- `updatedAt` ‚Äî set at creation and refreshed on every `updateNotebook` call.
- No default field values beyond `id`, `createdAt`, and `updatedAt`; the full `payload` is spread into the notebook record.

### 4. No Cascade Delete
`deleteNotebook` removes only the notebook entity. Notes that reference the deleted notebook via their `notebookId` field are **not** affected ‚Äî they remain in the database with their original `notebookId` value. Orphan handling is the responsibility of the UI layer or a dedicated cleanup operation.
